JoyMouse Project Development Guide for TaskMaster AI
1. Project Overview
JoyMouse is a comprehensive accessibility gaming solution designed for people with severe physical disabilities who can only use a mouse. It provides a transparent overlay virtual controller system that enables full game interaction through mouse-only input, featuring virtual joysticks and customizable buttons with hover-based activation.
Target Audience:

People with severe physical disabilities who can only use a mouse
Accessibility researchers studying adaptive gaming technologies
Assistive technology developers building inclusive solutions
Game developers interested in implementing inclusive design principles

Benchmarking References:

Alt Controller: GUI-based input mapping, profile structure, multi-function input mapping architecture
Flex Controller Expansion App (FCEA): Panel-based UI, dwell method, macro functionality, profile/panel set structure

2. Technical Stack & Architecture
Current Implementation Structure:
JoyMouse/
├── main.py                        # Entry point and application initialization
├── core/
│   ├── logic.py                   # Joystick logic and movement tracking
│   ├── app_manager.py             # Mode state management and signals
│   ├── mode_controller.py         # Window focus manager for game targeting
│   └── settings.py                # INI-based configuration parser
├── ui/
│   ├── button_widget.py           # Interactive buttons (hover/click modes)
│   ├── button_manager.py          # Button registry and JSON/INI config I/O
│   ├── button_editor.py           # Per-button settings dialog
│   ├── mode_toolbar.py            # Mode toggle interface
│   └── hover_button.py            # Legacy hover button implementation
├── output/
│   ├── gamepad_output.py          # vgamepad Xbox controller emulation
│   ├── keyboard_output.py         # pynput keyboard simulation
│   └── mouse_handler.py           # pynput mouse event simulation
├── config/
│   └── joystick_settings.ini      # Persistent joystick configuration
└── resources/
    └── icon.png                   # UI assets and icons
Technology Stack:

Framework: Python with PyQt5 for GUI and transparent overlays
Input Libraries:

vgamepad for virtual Xbox controller emulation
pynput for keyboard/mouse simulation


Configuration: INI files for joystick settings, JSON for button layouts
Graphics: Qt's QPainter for custom drawing, transparency, and animations

3. Core Features & Implementation Details
3.1 Virtual Joystick System (core/logic.py)
Current Implementation:

Circular transparent virtual joystick with always-on-top display
Mouse movement within defined radius controls analog stick input
Deadzone filtering to ignore unintentional minor movements
Free movement and resizing through drag operations
Visual feedback with configurable colors, line width, and transparency
INI-based persistence for all settings
Automatic position reset and input release when mouse exits window
Floating settings button for configuration access

Technical Specifications:

Real-time mouse tracking with circular area constraints
vgamepad left joystick data transmission
Configurable radius, deadzone, visual styling
Click-through mode planned for non-blocking mouse passthrough

3.2 Button System (ui/button_widget.py)
Button Types & Modes:

Activation Methods:

Hover activation with progress gauge visualization
Click interaction for immediate response


Behavior Modes:

Press: Single activation trigger
Holding: Sustained input while active
Toggle: On/off state switching


Advanced Features:

Drag repositioning and resize handles in edit mode
Double-click settings dialog access
Button duplication with ID-based management
Position offset when cloning buttons



Gauge Implementation:

Timer-based progress visualization (0.0-1.0 scale)
Visual feedback with color transitions and filling animations
Configurable dwell time and release time parameters
Fade-out or maintain options based on user settings

3.3 Input/Output Processing (output/ modules)
Architecture:

Modular output system with clear separation of concerns
Signal-based decoupled processing for responsive input handling
Support for multiple output types:

gamepad_output.py: vgamepad Xbox controller emulation
keyboard_output.py: pynput keyboard simulation
mouse_handler.py: pynput mouse event simulation



3.4 Mode Management (core/app_manager.py)
Dual Mode System:

Play Mode:

Real gamepad/keyboard/mouse input transmission
UI elements hidden or minimized for gameplay
Focus maintenance on target game windows


Edit Mode:

UI repositioning and customization enabled
Drag movement, resize handles, settings access
No input transmission to prevent accidental game actions



Focus Management (core/mode_controller.py):

Automatic game window detection (Steam, Unity, etc.)
SetForegroundWindow API for focus maintenance
Per-game profile loading based on focused application

4. Configuration & Data Management
4.1 Settings Architecture (core/settings.py)
Joystick Configuration (INI):
ini[Joystick]
radius = 100
deadzone = 0.1
line_width = 3
transparency = 0.8
color_active = #00FF00
color_inactive = #808080
Button Configuration (JSON):
json{
  "buttons": [
    {
      "id": "btn1",
      "type": "hover",
      "mode": "toggle",
      "position": {"x": 400, "y": 300},
      "size": {"width": 120, "height": 120},
      "output": {"type": "gamepad", "button": "XINPUT_A"},
      "timing": {
        "gauge_time": 1.2,
        "release_time": 0.3,
        "repeat": false
      },
      "appearance": {
        "label": "Jump",
        "color": "#40E0D0",
        "transparency": 0.7
      }
    }
  ]
}
